## 既知の不安定挙動と対処(再発時用)

### OpenClaw の `agent --local` で fetch-timeout

**症状**: `infer model run --local` は PONG 返るのに、`agent --local --agent main` で 12 秒のセットアップ後 `[fetch-timeout]` で abort。

**原因**: daemon 側 heartbeat embedded session が同一 `agent:main:main` の jsonl ロックを保持し続け、新規 CLI run が 10 秒のロック待ちタイムアウトに落ちて embedded fetch watchdog が発火。

**対処**:
```bash
launchctl bootout gui/501/ai.openclaw.gateway
mkdir -p ~/.openclaw/_quarantine/sessions-$(date +%s)
mv ~/.openclaw/agents/main/sessions/* ~/.openclaw/_quarantine/sessions-$(date +%s)/
launchctl bootstrap gui/501 ~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

### API キーが OpenClaw daemon に届かない

**症状**: `~/.openclaw/.env` には書いたのに daemon プロセス env に `OPENAI_API_KEY` が無い。

**原因**: LaunchAgent が読む env file は `~/.openclaw/service-env/ai.openclaw.gateway.env` のみで、`~/.openclaw/.env` は対象外。

**対処**: env wrapper script を編集して `.env` を source。
```sh
#!/bin/sh
set -eu
env_file="$1"
shift
if [ -f "$env_file" ]; then . "$env_file"; fi
if [ -f "$HOME/.openclaw/.env" ]; then
  set -a
  . "$HOME/.openclaw/.env"
  set +a
fi
exec "$@"
```

### Node 複数バージョン時の install path 不整合

**症状**: `openclaw doctor` が "Gateway service entrypoint does not match" と警告。

**対処**: `openclaw doctor --fix` で entrypoint plist を再生成。または `~/Library/LaunchAgents/ai.openclaw.gateway.plist` を直接確認し、Node binary path が現行 install と一致しているか目視。
