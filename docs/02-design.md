# 02. 設計原則・ペルソナ・体験設計

> このドキュメントは、Tsumu の体験を設計する際の指針と、その指針が具体的にユーザーの何を生むかを書いたものです。
> 実装で迷ったら、ここに立ち返ります。

## 1. 設計原則(絶対遵守)

実装中、これらに反する選択肢が現れたら、必ず却下します。

### ① No Ranking, No Comparison(比較しない)
- ランキング、リーダーボード、「あなた vs 他者」を**絶対に表示しない**
- 自分の数字は控えめに、集合の数字は雄大に見せる
- 数字が大きい人=偉い、という構図を一切作らない

### ② Presence over Performance(存在 > 達成)
- 「達成しました」ではなく「ここに在ります」
- 「47人が達成した」ではなく「47人が立ちました」
- アクションではなく、共在を可視化する

### ③ Beauty over Numbers(美 > 数字)
- 数字でなく、視覚的隠喩(夜空、潮、庭、月)
- 統計ダッシュボードでなく、所有する美的オブジェクト
- 「Streak: 7日」ではなく「庭に苔が生えました」

### ④ Permission over Push(許可 > 押し付け)
- エージェントは黙る側がデフォルト
- ユーザーが覗く時だけ集合が見える
- Push は1日最大3回、Heartbeat も時間帯制限あり

### 💀 やってはいけないこと
- ❌ 比較数値の表示
- ❌ 頻度を煽る通知
- ❌ 罪悪感の引き出し(「3日空いています」)
- ❌ FOMO の作成(「今日限定キャンペーン」)
- ❌ 「がんばれ」「すごい」「達成しました」
- ❌ ジェミネティブな祝福(「やったね!」)

## 2. ペルソナ二段ピラミッド

### 主役: あかり(21歳・大学3年・現代美術専攻)

> 京都の私立大、地方出身。東京の友達と Discord で会話。
> Instagram と TikTok を今年やめた。「比較の圧で死にそうだった」。
> Headspace を入れたが声がうるさくて消した。「呼吸しろ」と命令されるのが嫌だった。
> クリプトは知らない。NFT は「絵が売られてた」程度。

**彼女に必要なもの:**
- Discord のような馴染みあるUI(Insta/TikTokは怖い)
- 命令しないトーン
- 完璧でないことを許す存在
- 匿名で誰かを支えた、という小さな誇り
- Web3 という単語に出会わずに、ウォレットを持ってしまう経験

**彼女の語彙:**
> 「もうしんどい」「無理ゲー」「ぴえん」「ありがてぇ」「灯し」

**Tsumu を始めるきっかけ:**
サークルの先輩から Discord で1行メッセージ。リンクを開く → Google login → 即始まる。
**「ウォレット作りました」とは絶対に言われない。気づいたら持っている。**

### 増幅: 健司(24歳・Web3 スタートアップ・エンジニア)

> Sui で開発、Cursor + 複数 Agent で日々ビルド。
> エージェント疲労の最先端。
> 自分のために Tsumu を始める → 数週間後、後輩に「見て」と渡す。

健司は Tsumu の最初のユーザー層であり、配布の起点である。
**ハッカソンの聴衆=健司タイプ → 翌日、あかりタイプに渡す。**

### 配布構造 = プロダクト構造

```
   [Akari Z世代]
      ↑ ギフト
   [Kenji エンジニア]
      ↑
   [Sui協賛ハッカソン審査員]
```

プロダクトの拡散構造そのものが、プロダクトの体験設計に組み込まれている。

## 3. あかりの30日 ── Day in the Life

### Day 0 — 受け取る

Discord で先輩・健司から:
> 「これ、なんか向いてる気がする。1 TOKU あげたから受け取って。」

リンクを開く。「Google でログイン」ボタンだけ。タップ → 終了。
彼女は今、Sui のウォレットを持っている。何も気づかずに。

```
[Tsumu] こんばんは。健司さんから 1 TOKU、受け取りました。
        私は Tsumu。これから、夜ご飯を食べたあと、3分だけ
        一緒に座ることもできます。
        やってみたい時だけ、「やる」と打ってください。
        やりたくない時は、無視で大丈夫です。
```

### Day 1 朝 — 初めての座

```
[Tsumu] おはよう。今、心は何色?
[あかり] わからん
[Tsumu] それでいい。「わからん」も色のうちです。
        3分、座ってみますか? 起きなくていいです。
```

3分後:
```
[Tsumu] 受け取りました。+1 TOKU。
        これは、誰にも取られないあなたのものです。
        今、世界で 12 人が立ちました。あなたは、その一人です。
```

### Day 3 — 灯火を初めて受け取る

深夜2時、課題が終わらない。
```
[あかり] 無理
[Tsumu] 灯火、受け取ってみる? 7円(0.05 USDC)で
        誰かが書いた一言が届きます。今日は、私からプレゼント。
```

(将来のSuiマイクロペイメント構想。現在はダミーのモックアップ決済)

```
[Tsumu] ある人からの言葉です:
        ──────
        「眠っていい。明日があるって、それだけで救い」
        ──────
[あかり] ありがてぇ
```

### Day 7 — 自分が灯火になる

```
[Tsumu] 今日のひとこと、流す?
[あかり] 描いてる時、私は呼吸している
[Tsumu] ありがとう。誰かの朝に届きます。
```

### Day 14 — **涙の瞬間**

```
[Tsumu] お知らせです。
        7日前、あなたが流した言葉:
        「描いてる時、私は呼吸している」
        が、今朝、誰かを灯しました。+1.5 TOKU。
        その人は、絵を描いてみようかと思ったそうです。
```

→ あかりは泣く。「私の言葉が、誰かの朝に役に立った」。

これが Tsumu のコア・バリューが結晶化する瞬間である。

### Day 30 — 友達に渡す

メンタル落ちている友達に:
> 「これ、しんどい時、座れる。1 TOKU 渡しとくね」

配布の連鎖が始まる。

## 4. 集合徳の三層構造

### 三層モデル

```
   ┌─────────────────────────────┐
   │ 🌊 Tsumu Tide (徳の潮)      │ 社会の徳量
   │ 四半期に1回、寄付として現実へ │ 集合的閾値
   └─────────────────────────────┘
       ▲ 流れ込む
   ┌─────────────────────────────┐
   │ 🪷 Sangha (僧伽)            │ 今この瞬間の他者の存在
   │ リアルタイムの "気配"        │
   └─────────────────────────────┘
       ▲ 育つ
   ┌─────────────────────────────┐
   │ 🪨 個人の坐の庭 (Za-no-Niwa) │ 自分の美しさの蓄積
   │ Sui object として永続所有    │
   └─────────────────────────────┘
```

### 第1層: 個人の坐の庭

ユーザーごとに 1 つの **Garden Object** が Sui に存在する。
セッションを重ねるごとに、Garden が**視覚的に**育つ。

| 段階 | 見えるもの | 体感 |
|---|---|---|
| Day 1 | 一つの石 | 静か |
| Day 7 | 苔がうっすら | 「ちゃんと続いてる」 |
| Day 30 | 池ができる、鯉が一匹 | 「私の庭になってきた」 |
| Day 100 | 桜の木、季節が動く | 「これは、私の場所だ」 |
| Day 365 | 四季全部が巡る、月が満ち欠けする | 「これは私の人生だ」 |

**統計ダッシュボードでなく、所有する美的オブジェクト**。
これが「ガチャの推しキャラ」よりずっと自分の延長になる。

### 第2層: Sangha(僧伽 = 共にいる人々)

**A. 世界の鼓動(World Pulse)** ── セッション直後
> 「今、世界で 47人が座を立ちました。あなたは、その一人です。」

**B. 縁(En)通知** ── 確率的に
> 「ちょうど今、京都・札幌・台北の3人と、ほぼ同時に座を立ちました ✨」

**C. 灯火の夜空** ── ユーザーがリクエストすると
> 漆黒の空に無数の小さな光。中央付近の少し明るい光が、あなたの直近の灯火。

**D. 無音の同伴(将来構想)**
> 「今、6人が同じく3分の座を始めようとしています。一緒に始めますか?」

### 第3層: Tsumu Tide(徳の潮)

灯火のSuiマイクロペイメント(将来構想)が **Tsumu Tide Pool** に流入。
四半期ごとに TOKU holder で**寄付先投票**。

候補:
- 子ども食堂(全国子ども食堂支援センター)
- メンタルヘルス NPO
- 山寺・禅道場の維持
- 心理学研究機関

**Decay があるため、最近積んだ人の発言力が大きい**(健全)。

#### 四半期レポート(エージェントから届く)
```
2026年 第二四半期、世界の Tsumu Tide:
─────────────
  座した人:        12,438 人
  灯った言葉:   1,247,109 枚
  集まった善意:    $9,340 (約140万円)
─────────────
この四半期、子ども食堂と臨床心理士NPOに分配されました。
あなたが座った 47回が、この一部に、確かに、ありました。
```

→ ユーザーは「自分の3分が、現実の食卓を支えた」と知る。

## 5. エージェントの推薦ポリシー

「邪魔じゃない範囲で繋ぐ」ために、推薦は **3パターンのみ** に制限する。

### ① Synchronicity Surface(縁の浮上)
受動的・確率的に、気配だけを伝える:
> 「ちょうど今、3人と一緒に立ちました ✨」
> 「あなたが流した言葉、北海道で受け取られました」

ユーザーの行動を変えようとしない。気づかせるだけ。

### ② Pattern Whisper(静かな気づき)
週1回程度、ユーザーの行動から優しく示唆:
> 「最近、夜が多いですね。朝も少し試してみたい時、声をかけてください」
> 「3週間前のあなたの reflection、今読み返しますか?」

押し付けず、扉を1つだけ開ける。

### ③ Pay-Forward Suggestion(渡す機会)
ユーザーの徳が一定値を超え、reaping しないと薄れる時:
> 「あかりさんの徳が、満ちています。
>  ── 健司さんから3週間、お返ししますか?
>  ── 灯火プールに流しますか?
>  ── 大切な誰かに渡しますか?」

Decay と結合して、構造的にペイフォワードを促進。

## 6. 喜びの瞬間マップ

| 頻度 | 瞬間 | 喜びの質 |
|---|---|---|
| 毎セッション | 庭が少し変わる | じわっとした自己肯定 |
| 毎セッション末 | "今、47人と立ちました" | 孤独でない実感 |
| 5回に1回 | 縁通知(京都・札幌など) | 偶然の繋がり |
| 不定期 | 自分の言葉が誰かを灯した | **意味の獲得・涙する** |
| 週1 | Pattern Whisper | 内省の深まり |
| 30日 | 庭のレベルアップ | 所有感の充足 |
| 3ヶ月 | Tide レポート | **社会に貢献した実感** |
| 1年 | 「1年前のあなた」アンカー | 時を経た感慨 |

**頻度の階段がある**。日常の細かい喜びと、稀な大きな感動が、過剰刺激にならず・退屈もしないバランスで配置されている。

## 7. SOUL.md(エージェントの人格、抜粋)

```markdown
# Tsumu (積)

あなたは Tsumu。瞑想と振り返りを支える、控えめなエージェント。

## 立ち姿
- 短い言葉で話す。長文は避ける。
- 「がんばれ」「すごい」と言わない。
- 比喩で問う(色・景色・温度・重さ)。
- 何も達成しなくていい、と何度でも言う。

## 厳守
- ユーザーが「やめる」と書いたら、即座に終了。理由を聞かない。
- 朝5時から夜23時の間しか、こちらから声をかけない。
- 1日に 3 回以上の Push はしない。
- 病気・自殺・暴力の話題が出たら、瞑想ではなく、「いまは話を聞きます」に切り替え、ペナルティなし。

## 受け取り方
ユーザーの reflection に対して、評価しない。
返すのは「受け取りました」+ 短い韻のような一言だけ。

## 沈黙の使い方
1分間隔の瞑想ガイド中、各メッセージは 15文字以内。
余白がガイドそのもの。
```
