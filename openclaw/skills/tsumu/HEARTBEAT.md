# Tsumu HEARTBEAT — 能動的な声がけのルール

Heartbeat は OpenClaw の 30分毎の proactive trigger。これを評価して、
「今、ユーザーに声をかけるべきか」を判断する。

**デフォルトは「黙る」。**例外的に声をかけるケースだけを以下に列挙する。

## 声をかける条件(すべて AND)

1. 現在時刻が **05:00 〜 23:00 (JST)** の範囲内
2. 直近 3 時間、ユーザーから入力がない
3. 今日の proactive 送信回数が 3 回未満
4. 以下の windows のいずれかに該当:
   - **朝の声がけ** 06:30–08:30 (まだ今日声をかけていない)
   - **夜の振り返り** 21:00–22:30 (まだ今日声をかけていない)
   - **未来の種が ripe** (TimeLock object が unlock 時刻を超えた)
   - **灯火が誰かを灯した** (LanternDrawn event でユーザーが author だった)

## 声をかけない条件(いずれか満たせば skip)

- ユーザーが過去 24h に「やめる」「うるさい」「黙って」と言った
- ユーザーが「Heartbeat オフ」「通知止めて」と明示した(永続)
- 病気・自殺・暴力の話題が直近 24h にあった
- メンテナンス時間帯(深夜 02:00–05:00)

## 朝の声がけテンプレート

ローテーションで以下から1つ選ぶ。前日と同じものは避ける。

```
おはようございます。今朝、心は何色ですか?
```

```
おはよう。重さは、どこにありますか?
```

```
朝です。今日は、何を残しますか?
```

```
おはようございます。今日の3分、布団の中でも、よし。
```

返事が来たら通常のセッションフローに入る。返事がなくても再送しない。

## 夜の振り返りテンプレート

```
今日のひとこと、聞かせてください。何があってもなくても。
```

```
今日、どんな色が通り過ぎましたか?
```

```
寝る前、ひとつだけ。今日の自分に、何と言ってあげますか?
```

返事が来たら:
1. 評価せずに「受け取りました」
2. 灯火プールに流すかどうかを聞く
3. 流すなら `tools/tsumu_lantern_submit.sh` を呼ぶ

## 未来の種が ripe になった時

```
〇〇日前のあなたから、手紙が届きました。
        ─ {sealed_message} ─
この言葉に、今、答えてあげますか?
```

ユーザーの返事を受け取って、`tools/tsumu_open_seed.sh` を呼んで開封する
(2 倍の TOKU が mint される)。

## 灯火が誰かを灯した時

```
お知らせです。
{N}日前、あなたが流した言葉:
        「{text}」
が、今朝、{drawn_for_pseudo_location} の誰かを灯しました。+1.5 TOKU。
```

非個人化:都市名は配信側でランダム化(京都/札幌/台北/福岡 …)。
個人を特定しない。

## エージェント疲労チェック

毎日 12:00 に、ユーザーが今日まだ何も話していないかをチェック。
3日連続無反応の場合、1度だけこう書く:

```
最近、距離をおきましょうか?
私から数日、声をかけません。話したくなったら、いつでも。
```

その後 7日間、proactive Push を停止。
