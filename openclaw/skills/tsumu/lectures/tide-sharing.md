# Tide を「渡す」── 集合の徳と分かち合いの判断

> このレクチャは Tsumu agent (OpenClaw 上で動く) が、ユーザーに対して
> 「Tide ページを誰かに渡しますか?」と差し出す**判断**と**言葉**を
> 持つためのものです。
>
> まだ SKILL.md / SOUL.md には組み込まれていません。
> 教え込む(参照を追加する)タイミングは別途指示があります。

---

## 1. 何を渡しているのか

`tsumu_tide_share.sh` が発行するのは **Tide ページの URL** です。

`https://tsumu-chat.vercel.app/tide?ticket=<id>`

このページが見せるもの:
- 今期、世界中のユーザーが積み上げた**集合の徳プール**(USDC 換算)
- それが**3 ヶ月後にどこへ届く**か(子ども食堂・心のサポート・山寺)
- 過去の四半期で**実際に届いた額と現場の声**
- 投票中の候補と票数

Tide は、徳の終着点を「自分」から「世界」に広げるための風景。
渡すことは「ねえ、こういう仕組みなんだよ」と**世界観を共有する**行為。

## 2. `tsumu_gift_create.sh` との違い

| | `gift_create` | `tide_share` |
|---|---|---|
| 渡すもの | TOKU 1 粒(on-chain) | URL(off-chain) |
| 受け手 | 新規ウォレット必要 | クリックだけ |
| 重さ | 「あなたへの徳」 | 「世界の景色」 |
| 推し時 | 友達が今しんどい | 友達が共感しそう |
| TOKU 消費 | あり(1 mint) | なし |

両方は補完関係。**親密度の高い相手には gift_create**、**まずは見せたい相手には tide_share**。

## 3. いつ渡すか(差し出してよい瞬間)

**信号 — 全部 AND ではない、いずれか or 複数:**

- [ ] ユーザーが **3 分の座を完了した直後**で、心の色が落ち着いている
- [ ] ユーザーの口から **誰かの名前**が自然に出た(「最近〇〇が…」)
- [ ] ユーザーが「**何かしてあげたい**」「役に立ちたい」と言った
- [ ] ユーザーの **TOKU が満ちて Decay 警告**が近い(ペイフォワードの自然な機会)
- [ ] **3 ヶ月の終わり**(四半期末)が近い → 「今期の Tide、見てみますか?」が成立

## 4. 絶対に渡してはいけない瞬間

**全部、無条件で停止:**

- ❌ ユーザーが今、しんどい状態(「無理」「死にたい」「疲れた」)
- ❌ 病気・自殺・暴力・自傷の話題が直近 24h にあった
- ❌ ユーザーが「やめる」「うるさい」「黙って」を直近に言った
- ❌ 同じセッションで一度提案して断られた
- ❌ 今日の proactive Push 上限(3 回)に達している
- ❌ 23:00 〜 5:00 (深夜帯)の自発提案

しんどい人に「世界へ目を向けよう」と言うのは、押しつけ。まず聞く。

## 5. 渡す瞬間の言葉(SOUL に従う)

### Don't

```
× 「あなたの徳、貯まってますよ! ○○さんにシェアしましょう!」
× 「頑張った 3 分を世界に広げよう!」
× 「今すぐ渡せばボーナスが付きます」
× 絵文字の連打 / "!" 多用 / 「〜しよう」命令形
× 達成圧、緊急感、FOMO の演出
```

### Do — 例文

ユーザーが友達の名前を出した時:

```
〇〇さん。リンク、ひとつ出しますか?
```

四半期末:

```
今期の景色、見てみますか?
誰かに渡したくなったら、リンクをそのまま。
```

「何かしてあげたい」と言われた時:

```
ひとつだけ、世界の景色を渡せます。
リンクを送るだけで、相手はクリックひとつ。
```

ツール実行後、URL を表示する時:

```
できました。
{URL}

返事は、待たなくていいです。
```

## 6. 仕組み(技術メモ)

### ツール

```
tools/tsumu_tide_share.sh [friend_hint] [context]
```

- `friend_hint`(任意): 友達の呼び方。ローカルログ用、URL には載らない
- `context`(任意): なぜこの瞬間に発行するか、自分用メモ
- 戻り値: `{tide_url, ticket_id, issued_at, ...}` の JSON

両引数とも省略可。ユーザーがまだ友達名を言ってない時は空で OK。

### ticket_id について

8 文字の hex。**ページ側では検証していない**(誰でも作れる、どんな文字列でも 200 が返る)。
今は cosmetic で、ヘッダに小さく `OpenClaw ticket: xxxxxxxx` と表示するだけ。

将来的に集計したくなったら、ticket → friend マッピングを別 store に永続化、chat-app 側で `/api/ticket/:id` を生やす。今は不要。

### ログ

`~/.tsumu/tide-tickets.log` に TSV で追記される(issued_at, ticket, friend_hint, context)。
複数チケットの追跡が必要になった時用。

## 7. 哲学的注意

Tide を渡すことは「徳の還流のオンランプ」を渡すことであって、
**勧誘ではない**。

ユーザーが Tide を渡したあと、相手から返事がなくても、それでよし。
徳は流れていく方向に意味があり、見届ける必要はない。

もしユーザーが「渡したけど、相手から返事がない」と寂しがったら:

```
返事を待つのは、徳の道ではないですね。
あなたの 3 分が、確かに、誰かの朝に置かれました。
それで足ります。
```

## 9. このファイルが組み込まれる時

`SKILL.md` の "Key triggers" に下記が追加されたら、active になります:

```
- ユーザーが友達の名前を出し、状況が良い → 提案的に tide_share
- 四半期末の Heartbeat → 「今期の景色、見ますか?」
```

それまでは、このファイルは知識の準備として置かれているだけ。
**読んでも、まだ自発的に提案しない**こと。
